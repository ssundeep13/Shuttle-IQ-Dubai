New task — Always show players their next game (queued suggestions + improved Claude AI)
What & Why
Players on the waiting screen currently see "Finding your next game..." when all courts are occupied. They should always see their upcoming game assignment immediately. With 6 players and 1 court, the 2 waiting players should see their next game calculated from the current court players combined with themselves. With 24 players and 4 courts, all 8 waiting players should see their next game simultaneously with Claude AI deciding the optimal rotation across all courts at once.

Schema changes
Add two things to match_suggestions:

queued as a valid status value — update inline comment only, no migration needed
includesActivePlayers: boolean column defaulting to false — run npm run db:push and report output


Updated tryAutoMatchmaking — queued suggestion creation
After filling available courts with pending suggestions, loop over occupied courts:
FOR each occupied court with no queued suggestion:

  CASE 1: waiting players >= 4
  → run matchmaking using waiting players only
  → create queued suggestion
  → status = 'queued', pendingUntil = null, includesActivePlayers = false

  CASE 2: waiting players >= 1 
  AND (waiting players + players on this court) >= 4
  → run matchmaking using ALL players:
    (waiting players + 4 players currently on this court)
  → create queued suggestion
  → status = 'queued', pendingUntil = null, includesActivePlayers = true

  CASE 3: waiting players = 0
  → exit

Claude AI vs standard algorithm
IF queued suggestions needed for 1 court:
  → use generateBracketedLineups

IF queued suggestions needed for 2+ courts simultaneously:
  → use Claude AI (once for all courts)
  → 5 second timeout, fall back to generateBracketedLineups if fails
